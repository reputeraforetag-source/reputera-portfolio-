## üõ†Ô∏è Prompt Engineering Methodology

My approach follows a structured 5-step framework:


1. ROLE DEFINITION ‚Üí 2. CONTEXT SETTING ‚Üí 3. CONSTRAINTS ‚Üí 
4. STRUCTURE ENFORCEMENT ‚Üí 5. VALIDATION CRITERIA
Example: Before vs After
Before - Basic Prompt:


"Help me fix this API error"
After - Professional Prompt:


You are a senior principal engineer with 20 years experience debugging 
distributed systems. Diagnose this 500 error in our production API:

SYMPTOM: [Specific error]
EXPECTED: [Expected behavior]
ENVIRONMENT: [Tech stack details]
ERROR LOGS: [Relevant logs]

Follow this methodology:
1. Root cause hypothesis (list 3 most likely causes)
2. Diagnostic steps (exact commands to run)
3. Isolation procedure
4. Fix implementation
5. Regression prevention
